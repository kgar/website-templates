Responsive Breakpoints

The Main Content

This demo deals with the Responsive Breakpoints of a responsive website. These breakpoints help to control the flow of content on various screen sizes. The simplest implementation of this involves media queries based on the screen width. If you resize the browser window, you will see the breakpoints trigger in realtime.
This demo uses a mobile-first approach. That is, if I removed all of the media queries, then the design would be as it would be on a mobile device. For this reason, the media queries deal with the min-width rather than the max-width. The media queries stack up their styles as the browser window becomes larger.

The nice thing about mobile-first design is that you can focus on content and simplicity of design when initially working on your website. Is this the intention of mobile-first design? Nah. But it does ice the cake.
